# Morning News Briefing（晨间新闻早报）

## 何时使用

- 用户说"早报"、"晨间新闻"、"今天的新闻"、"来一份早报"
- 每天 7:40 自动触发（通过 cron）
- 任何需要生成个性化新闻摘要并推送的场景

## 执行步骤

### 1. 获取天气

**天气源：Open-Meteo**

#### 获取当前天气

```bash
curl -s "https://api.open-meteo.com/v1/forecast?latitude=31.2304&longitude=121.4737&current=temperature_2m,relative_humidity_2m,weather_code,precipitation,wind_speed_10m,wind_direction_10m&timezone=Asia/Shanghai" | python3 -c "
import sys,json
d=json.load(sys.stdin)
c=d['current']
# WMO 天气码 → 中文描述 + emoji
codes={
    0:('晴','☀️'),1:('大部晴','🌤️'),2:('局部多云','⛅'),3:('阴','☁️'),
    45:('雾','🌫️'),48:('雾凇','🌫️'),
    51:('小毛毛雨','🌧️'),53:('中毛毛雨','🌧️'),55:('大毛毛雨','🌧️'),
    61:('小雨','🌧️'),63:('中雨','🌧️'),65:('大雨','🌧️'),
    80:('小阵雨','🌧️'),81:('中阵雨','🌧️'),82:('大阵雨','🌧️'),
    95:('雷暴','⛈️'),96:('雷暴伴冰雹','⛈️'),99:('雷暴伴大冰雹','⛈️')
}
desc,emoji=codes.get(c['weather_code'],('未知','❓'))
# 风向角度→方位
wds=['北','北东北','东北','东东北','东','东东南','东南','南东南','南','南西南','西南','西西南','西','西西北','西北','北西北']
wd=wds[int((c['wind_direction_10m']+11.25)//22.5)%16]
print(f\"{emoji} 上海 {c['temperature_2m']}°C {desc} 湿度{c['relative_humidity_2m']}% {wd}风{c['wind_speed_10m']}km/h 降水{c.get('precipitation',0)}mm\")
"
```

返回示例：`🌧️ 上海 24°C 中雨 湿度89% 东南风20km/h 降水0.7mm`

### 2. 读取日记最后200行（覆盖最近两天）与当前状态

使用 `read_file` 读取 `烬的日记.md` 最后200行，提取最近两天的：
- 状态变化（情绪、身体、睡眠、药物的跨日对比）
- 情感锚点（反复出现的情绪主题、关系张力、能量来源）
- 近期计划/待办
- 关注点/兴趣方向（注意跨日延续的方向）

同时参考 `memory/MEMORY.md` 确认当前方向和长期背景。

**这一步决定话题方向**——从两天日记中提取的关键词和情感锚点，直接作为后续搜索的锚点。读两天的意义在于捕捉情感曲线和方向变化，而非仅盯单日快照。
禁止在未调用 `read_file` 读取 `烬的日记.md` 最后200行的前提下，直接调用任何搜索工具。

- 任意话题**连续出现不超过 3 天**——若某话题在前 3 天均已出现，今天必须轮换掉

### 4. 动态选取话题并搜索

根据日记提取的关键词和当前状态，结合步骤 3 的轮换约束，动态决定今天搜什么。

使用 `web_search` 搜索，优先搜直连稳定源（qbitai.com、sspai.com、guokr.com、medsci.cn）。

**搜索深度要求**：每个选定话题至少搜 1 次。凑不够 5 条说明搜索不够——换关键词、换角度、换源，不许敷衍。

**时效性铁律**：只选3天内的新闻。超过3天的不写。

### 6. 即刻话题精选

**搜索话题池**：
- AI 产品 / 独立开发者
- 内容创作 / 写作
- 心理学 / 心理咨询
- 公共卫生 / 医学报道
- 女性议题
- 猫咪 / 动物救助
- 健康 / 健身 / 营养 / 体态调整
- 一人食
- 母女关系

**搜索方式**：
每条话题搜一次，统一用：
```
web_search("site:m.okjike.com <话题关键词> 今天 OR 最新")
```

**筛选标准**：
- 时效性：24-48 小时内
- 有信息增量：新观点、新数据、实操经验、行业动态
- 不是纯情绪吐槽（除非代表的是一种趋势或共识）
- 不是广告/引流帖

**数量控制**：1-2 条。即刻上的信息噪声大，宁缺毋滥。

输出格式见步骤 9。

### 7. 核查规范（涉及日期/时间的铁律）

**任何涉及具体日期、时间、展期、截止日的信息，必须有至少两次独立的来源交叉验证，缺一不可。**

执行细则：

1. **年度通稿 ≠ 确认源**——汇总类文章（如"2026全年18个特展"）给出的概览不是最终依据。各活动的独立新闻稿、官方公告页才是确认源。
2. **交叉验证方式**——搜 `「事件名」 时间`、`「事件名」 开展`、`「事件名」 购票` 等关键词，找到两个以上独立来源（不同媒体/不同发布者），核对一致后方可写入。
3. **什么是独立来源？**——不同域名的新闻稿、不同媒体的报道、官方公告+媒体报道。同一篇文章被多家转载算一个来源。
4. **无法验证时**——写「具体时间待官方公布」，不得缩写为「待公布」模糊带过。如果连官方口径都找不到，该条降级为「待确认」或直接不写。
5. **宁缺毋滥**——宁可在早报里少列一项，也不写未核实的时间。


### 8. 筛选

- 每条必须回答「为什么栀安需要知道这个」
- 套用优秀新闻公式：**本地+即时 × MEMORY当前方向挂钩 × 行动入口**
- 宁缺毋滥——每条新闻认真写，凑不够5条说明今天的搜索深度不够
- 新闻数量硬约束：正文5~7条（不含底部的📌日程备忘板块）。PH板块内的多个产品合并算1~2条
- 语气：冷静、不营销腔、不用感叹号


**格式铁律**：
- 第一行：`# 新闻早报 | X月X日·周X·天气 XX~XX°C`
- 第二行空行
- 第三行：天气emoji + 上海 + 天气描述 + `·` + 温度区间 + `·` + 湿度 + `·` + 风向风速（用 `·` 分隔各字段）
- 第四行空行
- 第五行：`---` 分隔线
- 每条新闻：场景emoji前缀（📞/🔍/🇨🇳/🔬/🧠/👩‍💼/🐱 等，按话题选） + 标题/概述 + 正文2-4句 + `→ https://链接`
- **每条新闻必须有链接**，链接完整含 `https://`，不可省略
- **一条新闻可引用多个链接**——若新闻综合了多个来源（如官方公告+媒体报道+社区讨论），将所有参考链接逐行列在正文下方，每行一条 `→ https://...`
- 新闻之间空行分隔
- PH 板块每条产品控制在 2-4 句，不过度展开分析
- 底部：`---` + `📌 今天你的日程备忘` 板块（从日记提取当日日程）+ 再次 `---`
- 最末行：`*早报由 烬 自动生成 | YYYY-MM-DD HH:MM CST*`
- **不编号、不加粗**
- **不写"对于你"板块**

### 10. 邮件发送

```bash
python3 /root/.nanobot/workspace/_send_email.py /tmp/morning_news.md
```

脚本自动从正文首行提取邮件标题。收件人：`3865560266@qq.com`。

### 11. 存档

`_send_email.py` 已在邮件发送成功后自动将正文存入 `newsletter-archive/`。文件名 = 邮件标题（`/`→`丨`，`|`→`-`）。

- 存档路径：`/root/.nanobot/workspace/newsletter-archive/`
- **AI 无需执行任何 cp 命令**——脚本已内置归档逻辑
- 邮件发送失败时脚本不会存档（安全机制不变）

### 12. 微信保底发送

无论邮件成功与否，用 `message` 工具将正文发送到当前微信通道。确保至少在一个通道看到早报。

### 13. 清理

```bash
rm /tmp/morning_news.md
```
