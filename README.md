# 晨间新闻早报 · Morning News Briefing
本项目不仅是新闻抓取，更是对“主动式Agent”工作流的一次实战探索，旨在验证大模型在长链条任务中的指令遵循能力。

> **AI 落地的完整闭环：从"卖工具"到"直接交作业"。**

---

## 一句话

每天 7:30，一份**基于你真实生活轨迹**的个性化新闻早报，通过邮件和微信双通道自动送达。

不是 RSS 聚合，不是 AI 摘要。它是**把 AI 嵌进用户每日信息流的交付产品**。

---

## 为什么不是又一个新闻工具？

| 传统方式 | 这个晨间早报 |
|---------|---------|
| 用户自己订阅报纸、筛选关键词 | AI 从用户日记中提取关注点 |
| 固定频道，千人一面 | 每天的选题都不一样，跟着用户状态走 |
| 推送完就结束 | 每一条都给出"为什么与你有关"和行动入口 |
| 需要用户打开 App | 邮件 + 微信，AI主动向用户已在的通道推送新闻早报 |

2026 年 5 月 12 日，OpenAI 成立交付公司，官方表态：**"模型够好了，卡点在交付。"** 这个项目回应了同一个命题——不做更好的模型，做更好的交付。

## 工作流

> 完整技能文档见 [`SKILL.md`](SKILL.md)（353 行，包含话题轮换、核查规范、PH 四层容错等全部细节）。

```
用户日记（每日更新）
      ↓
  提取关键词 + 当前关注点
      ↓
  动态选题（不是固定赛道，是从生活轨迹里长出来）
      ↓
  web_search × N（Tavily 引擎）
      ↓
  筛选：本地+即时 × 用户方向挂钩 × 行动入口
      ↓
  按固定格式写入 Markdown
      ↓
  ┌─ 邮件发送（SMTP，QQ邮箱）
  └─ 微信保底（Nanobot 消息通道）
```

---

## 技术栈

| 层 | 选型 |
|---|------|
| AI 引擎 | DeepSeek V4 Pro（路由决策 + 内容生成） |
| 搜索 | Tavily Search API |
| 天气 | Open-Meteo API（免费，无 Key） |
| 调度 | Nanobot cron（CST 时区） |
| 邮件 | QQ 邮箱 SMTP + Python 脚本 |
| 微信 | Nanobot WeChat Channel |
| 记忆系统 | Markdown 日记 + MEMORY.md + Dream 压缩机制 |
| 运行环境 | 腾讯云 Ubuntu 4GB + Python 3.12 |

**总成本：约 ¥30/月**（API token），不含人工/服务器。
