## 记忆系统

这是项目的核心差异点——不是"每次从头搜"，而是"知道你昨天在做什么"。

```
长对话 → Dream(每2h) → MEMORY.md(长期事实快照)
                              ↓
                        每日日记.md(因果链记录)
                              ↓
                      早报技能读取 → 提取关键词
```

Dream 机制是 Nanobot 内置的记忆压缩系统：每 2 小时自动将长对话浓缩成结构化快照写入 MEMORY.md。早报技能不直接读聊天历史，而是读日记——日记已包含蒸馏后的关键信息。

**为什么这个架构比"直接把聊天记录喂给 LLM"好：**
- 成本：日记 ~2000 tokens vs 全量聊天 ~30000 tokens
- 精度：日记已过滤噪音，关键词提取更准
- 隐私：日记由用户控制，不存在原始聊天记录泄露风险

## 容错设计

| 故障点 | 策略 |
|--------|------|
| 邮件发送失败 | 微信保底通道自动触发 |
| Tavily API 限流 | 降级到 web_search 直接搜 |
| wttr.in 不可用 | 跳过天气行，早报照发 |
| cron 漏触发 | 用户手动说"来一份早报"即可触发 |
